# AWS를 이해하기 위한 기초지식: OSI 7 Layer Model (1) – Physical/Data Link

이 문서는 OSI 7 계층 모델의 하위 두 계층인 물리 계층(Physical Layer)과 데이터 링크 계층(Data Link Layer)의 기본 개념과 역할을 설명합니다.

## 1. OSI 7 Layer Model (OSI 7 계층 모델)

- **정의**: 컴퓨터 네트워크와 통신을 7개의 추상적인 계층으로 나누어 표현한 모델입니다.
- **구조**:
    - 총 7개의 계층으로 구성됩니다.
    - 각 계층은 하위 계층의 기능을 활용하여 역할을 수행하고, 그 처리 결과를 상위 계층으로 전달합니다.
    - 낮은 계층부터 높은 계층 순서로 동작합니다.
- **설명 범위**: 이 강의에서는 Ethernet 관련 내용을 중심으로 설명합니다.

## 2. Physical Layer (물리 계층 - Layer 1)

- **정의**: 장치를 연결하기 위한 매체의 물리적인 사양을 정의하는 계층입니다.
    - 예: 전압, 주기, 시간, 전선 규격, 통신 거리 등
- **주요 단위**: **Bits (비트)**
- **대표 구성 요소**:
    - **전송 매체**: 케이블, 안테나, RF (무선 주파수), 광섬유, 구리선, USB 등
    - **네트워크 장비**: 허브(Hub), 리피터(Repeater)
- **Hub (허브)**:
    - 물리 계층에서 여러 기기를 연결하는 장치입니다.
    - 받은 내용을 그대로 모든 포트로 전달(Broadcast)합니다.
    - 에러/충돌(Collision) 감지나 디바이스별 제어 기능이 없습니다.
- **물리 계층에서 해결하지 못한 문제**:
    - **충돌**: 여러 장치가 동시에 데이터를 보내면 충돌이 발생합니다.
    - **대상 지정 불가**: 항상 모든 장치에 브로드캐스트하므로 특정 대상을 지정하여 전달할 수 없습니다.

## 3. Data Link Layer (데이터 링크 계층 - Layer 2)

- **정의**: 물리적인 통신을 제어하여, 직접 연결된 두 장치 간의 통신과 데이터 전송을 안정화하기 위한 프로토콜을 정의하는 계층입니다.
- **주요 단위**: **Frame (프레임)**
- **주요 구성 요소**:
    - **Mac Address (맥 주소)**
    - **Switch (스위치)**
- **주요 특징**:
    - **CSMA/CD (Carrier-Sense Multiple Access with Collision Detection)**: 통신 매체를 감지하고 충돌을 탐지하여 각 장치 간의 통신을 원활하게 조율하는 방식입니다.
    - **주소 지정**: 대상을 구별하여 특정 장치에만 데이터를 보내는 **유니캐스트(Unicast)** 통신을 지원하며, **브로드캐스트(Broadcast)**도 지원합니다.

### MAC (Media Access Control) Address

- **정의**: 네트워크 인터페이스(NIC)에 부여된 고유한 하드웨어 주소입니다.
- **목적**: 데이터가 지정한 대상에게 정확하게 전달될 수 있도록 대상 장치를 식별하는 데 사용됩니다.
- **구조**: 48비트(6바이트) 주소이며, 보통 16진수 2자리씩 6개로 나열하여 표기합니다 (예: `00:1A:2B:3C:4D:5E`).
- **구성**:
    - **OUI (Organizationally Unique Identifier)**: 앞 3바이트. 제조사에 부여된 고유 식별자입니다.
    - **NIC (Network Interface Controller)**: 뒤 3바이트. 각 네트워크 인터페이스별 고유 번호입니다.
- **특징**:
    - 전 세계적으로 MAC 주소는 고유하며, 변하지 않는 값입니다. (IP 주소는 변동 가능)

### Frame (프레임)

- **정의**: 데이터 링크 계층의 데이터 단위입니다.
- **구조**:
    - **Preamble / SFD**: 프레임의 시작을 알립니다.
    - **대상 MAC 주소 (6 bytes)**
    - **소스 MAC 주소 (6 bytes)**
    - **Ethertype (2 bytes)**: 상위 계층(Layer 3)의 프로토콜 종류를 명시합니다.
    - **Payload (46~1500 bytes)**: 실제 데이터 (네트워크 계층의 패킷).
    - **Frame Check Sequence (4 bytes)**: 오류 검증을 위한 값입니다.

- **데이터 링크 계층에서 해결하지 못한 문제**:
    - **로컬 네트워크 한정**: 로컬 네트워크(동일 네트워크 대역) 외부로는 통신이 불가능합니다.

이 문서는 AWS 클라우드 네트워킹의 기초가 되는 물리적 및 논리적 데이터 전송 원리를 이해하는 데 도움을 줍니다.
