# 05. AWS 기초 지식: OSI 7 Layer - 나머지 계층 (5, 6, 7)

이 문서는 OSI 7 계층 모델의 상위 3개 계층인 세션(Session), 표현(Presentation), 애플리케이션(Application) 계층의 역할과 기능을 설명합니다.

## "나머지 레이어" - 소프트웨어 계층

-   **OSI 모델과 TCP/IP 모델**: 현대의 TCP/IP 모델에서는 OSI의 세션, 표현, 애플리케이션 계층을 하나의 **애플리케이션 계층**으로 통합하여 다룹니다. 이들은 하드웨어보다는 소프트웨어 수준에서 동작합니다.

## Session Layer (세션 계층 - Layer 5)

-   **정의**: 통신 장치 간의 연결(세션)을 수립하고 유지하는 방법을 정의합니다.
-   **과거의 역할**: 여러 터미널이 하나의 메인프레임에 접속할 때, 각 사용자를 구분하고 연결을 지속시키는 역할을 했습니다.
-   **현대의 역할**: HTTP의 **쿠키(Cookie)** 와 같이, 사용자의 상태를 기억하고 로그인 세션을 유지하는 등의 현대적인 세션 관리 메커니즘이 이 계층의 개념을 포함합니다.
-   **참고**: FTP와 같은 일부 프로토콜은 세션 계층을 자체적으로 구현하지 않습니다.

## Presentation Layer (표현 계층 - Layer 6)

-   **정의**: 수신된 데이터를 해석하고 애플리케이션 계층에서 사용할 수 있는 형식으로 변환하는 방법을 정의합니다.
-   **주요 기능**: 데이터 **파싱(Parsing)**, **압축 해제(Decompression)**, **복호화(Decryption)** 등을 담당합니다.
-   **예시**:
    -   바이너리(binary) 형태의 데이터를 `String`이나 `Integer`와 같은 프로그래밍 언어의 데이터 타입으로 파싱합니다.
    -   `gzip`으로 압축된 HTTP 본문(Body)을 압축 해제합니다.
    -   암호화된 데이터를 복호화하여 애플리케이션이 읽을 수 있도록 합니다.
    -   `Content-Type: application/json`과 같은 헤더 정보를 보고, 데이터를 JSON 형식으로 해석합니다.

## Application Layer (애플리케이션 계층 - Layer 7)

-   **정의**: 최종적으로 수신된 데이터를 **처리**하는 방법을 정의합니다. 즉, "데이터를 가지고 무엇을, 어떻게 처리할지"에 관한 계층입니다.
-   **역할**: 사용자에게 보이는 인터페이스와 직접 상호작용하며, 데이터를 기반으로 실제 로직을 수행합니다.
-   **HTTP 프로토콜 예시**:
    -   **Method**: `GET`, `POST`, `PUT`, `DELETE` 등 요청의 종류를 정의합니다.
    -   **Status Code**: `2xx`(성공), `4xx`(클라이언트 오류), `5xx`(서버 오류) 등 응답 상태를 나타냅니다.
    -   **Header**: `Host`, `User-Agent`, `Authorization`, `Content-Type` 등 요청/응답의 부가 정보를 포함합니다.
-   **AWS Application Load Balancer (ALB)**:
    -   애플리케이션 계층(Layer 7)에서 작동하는 대표적인 서비스입니다.
    -   HTTP 헤더, 호스트 이름(`web.mydomain.com`), 경로(`dashboard.mydomain.com`) 등을 분석하여, 요청을 적절한 대상 그룹(웹서버, 이미지 서버, Lambda 함수 등)으로 라우팅하는 **지능적인 트래픽 분산**을 수행합니다.

이 문서는 AWS 환경에서 데이터가 최종적으로 어떻게 해석되고 처리되는지, 그리고 ALB와 같은 서비스가 어떻게 애플리케이션 수준에서 동작하는지에 대한 기초 지식을 제공합니다.
