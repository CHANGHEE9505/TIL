# 09. AWS 기초 지식: RDBMS vs NoSQL

이 문서는 현대 애플리케이션에서 사용되는 두 가지 주요 데이터베이스 유형인 **RDBMS(관계형 데이터베이스 관리 시스템)**와 **NoSQL**의 핵심적인 차이점을 비교하고 설명합니다.

## RDBMS (Relational Database Management System)

-   **정의**: 데이터 간의 **관계**에 집중하여, **정형화된 형식**으로 데이터를 관리하는 DB 관리 시스템입니다.
-   **데이터 저장 방식**: 행(Row)과 열(Column)으로 구성된 2차원 **테이블**을 기반으로, 정해진 **스키마(Schema)** 형식에 따라 데이터를 저장합니다.
-   **데이터 처리**: **ACID** 원칙을 따르는 트랜잭션(Transaction) 처리를 통해 데이터의 정합성과 신뢰성을 보장합니다.
-   **쿼리**: **SQL(Structured Query Language)**을 사용하여 `JOIN` 등 테이블 간의 관계를 활용한 복잡한 쿼리 처리가 가능합니다.
-   **확장성**: 주로 수직적 확장(Scale-up, 서버 자체의 성능을 높이는 방식)으로 확장하기 때문에 수평적 확장이 어렵습니다.
-   **주요 AWS 서비스**: **Amazon RDS**, **Amazon Aurora**, **Amazon Redshift**
-   **주요 사용 사례**: 데이터 구조가 명확하고 일관성이 중요한 대부분의 애플리케이션에서 메인 DB로 사용됩니다.

### ACID란?

트랜잭션(Transaction)이 안전하게 수행되는 것을 보장하기 위한 4가지 성질입니다.

-   **원자성 (Atomicity)**: 트랜잭션의 모든 작업이 전부 성공하거나 전부 실패해야 합니다 (All or Nothing).
-   **일관성 (Consistency)**: 트랜잭션 수행 후에도 데이터베이스는 항상 올바르고 일관된 상태를 유지해야 합니다 (예: 재고가 음수가 되지 않음).
-   **독립성 (Isolation)**: 하나의 트랜잭션이 수행 중일 때, 다른 트랜잭션이 끼어들어 영향을 주지 않도록 독립적으로 수행되어야 합니다.
-   **지속성 (Durability)**: 트랜잭션의 수행 결과는 영구적으로 데이터베이스에 반영되어야 합니다.

## NoSQL (Not Only SQL)

-   **정의**: 정해진 스키마 없이, 다양한 형식의 데이터를 유연하게 처리하는 데이터베이스 시스템입니다.
-   **데이터 저장 방식**: 별도로 정해진 스키마가 없어 유연하게 데이터를 저장할 수 있습니다.
    -   **Key-Value Stores**: `Key-Value` 쌍으로 데이터를 저장합니다. (예: `laungauge: kr`)
    -   **Document Database**: `JSON`과 유사한 문서(Document) 형식으로 데이터를 저장합니다.
    -   **Graph Database**: 데이터 간의 연결(관계)을 중심으로 데이터를 저장합니다. (예: 소셜 네트워크)
-   **데이터 처리**: **BASE** 원칙을 기반으로 처리하여, 가용성과 확장성에 중점을 둡니다.
-   **쿼리**: 대부분 SQL 사용이 불가능하며, 복잡한 `JOIN` 없이 간단한 처리만 가능합니다.
-   **확장성**: 분산 처리가 용이하여 수평적 확장(Scale-out, 서버 수를 늘리는 방식)에 매우 뛰어납니다.
-   **주요 AWS 서비스**: **Amazon DynamoDB**, **Amazon MemoryDB** (Redis, Memcached), **Amazon DocumentDB**
-   **주요 사용 사례**: 캐시, 간단한 데이터 저장, 추천 로직 등 각 DB 구조에 맞는 특정 목적에 주로 사용됩니다.

### BASE란?

ACID보다 느슨하고 유연한 모델로, 가용성에 중점을 둡니다.

-   **Basically Available (가용성)**: 데이터는 여러 스토리지에 저장되어 항상 접근 가능하게 유지됩니다.
-   **Soft-State (느슨한 상태)**: 데이터베이스의 상태가 확정된 상태로 유지되지 않고, 시간이 지나면서 자연스럽게 원하는 상태로 수렴합니다.
-   **Eventual Consistency (최종적 일관성)**: 일관성이 바로 확보되지 않고, 시간이 지나 언젠가는 일관적인 상태로 수렴합니다.

## 정리: RDBMS vs NoSQL

| 항목 | RDBMS | NoSQL |
| :--- | :--- | :--- |
| **데이터 형식** | 정해진 스키마로 데이터 관리 | 자유로운 데이터 형식 |
| **데이터 처리** | ACID (엄격한 정합성) | BASE (가용성 중심) |
| **확장성** | 확장 어려움 (수직적) | 비교적 쉬움 (수평적) |
| **쿼리 복잡도** | 복잡한 쿼리 수행 가능 (SQL, JOIN) | 비교적 간단한 쿼리만 수행 가능 |
| **AWS 서비스** | RDS, Aurora, Redshift | DynamoDB, DocumentDB, MemoryDB 등 |

이 문서는 AWS 환경에서 워크로드의 특성에 따라 적절한 데이터베이스(RDS 또는 DynamoDB 등)를 선택하는 데 필요한 기초 지식을 제공합니다.
